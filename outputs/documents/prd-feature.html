Product Requirements Document
Smart Re-engagement Nudge
Draft v0.3 PM: Ishaan D.

Overview

Implement a personalised in-app nudge system that prompts dormant users (no activity ≥7 days) to complete their highest-value incomplete action. The nudge is triggered by ML model output, uses dynamic copy, and targets the specific screen where the user last dropped off.

Problem

28% of activated users go dormant within 14 days of signup. Current re-engagement is limited to weekly email blasts with generic CTAs — open rate 9%, click-through 1.2%. No in-app mechanism exists for users who return but don't know what to do next.

User Stories

As a returning user
I want to see a contextual prompt telling me what to do next, so I can pick up where I left off without hunting through menus.
AC: Nudge appears within 3s of app open if dormancy criteria met.
As a power user
I want to be able to dismiss nudges permanently for actions I've decided to skip, so I'm not pestered repeatedly.
AC: Dismiss stores preference; same nudge-type not shown for 30 days.

Functional Requirements

Must Have

  • Dormancy trigger: ≥7 days inactive
  • 3 nudge types: complete profile, first transaction, invite contact
  • Deep-link to relevant screen
  • Dismiss + snooze (24h / 7d) options
  • A/B test framework hooks

Nice to Have

  • ML-ranked nudge selection (v2)
  • Personalised copy using user name + action
  • Animated nudge card (Lottie)
  • Nudge performance dashboard for PM
