Product Requirements Document

Automated Challan Filing System

Version: v1.0
Date: 12 March 2025
Status: Draft
Author: Product Team

Problem Statement

Hub managers spend 45+ minutes daily manually filing traffic challans across 42 buses, leading to missed filings, duplicate payments, and significant compliance risks. Current process is entirely manual with no system-level tracking or alerts.

User Story

"As a Hub Manager, I want challans to be automatically detected and filed from the GPS system, so that I can eliminate manual tracking errors and reduce filing time to near-zero."

Success Metrics

Metric Current State Target
Challan filing time per hub 45 min/day <5 min/day
Missed challan rate 18% <2%
Duplicate filing incidents 4 / month 0 / month

Functional Requirements

  • 1 System must auto-detect challan from VAHAN API when bus number plate is queried.
  • 2 System must auto-populate challan details (amount, date, violation type) in iLens.
  • 3 Hub Manager must receive WhatsApp notification for each detected challan within 15 minutes of detection.
  • 4 One-click filing and payment confirmation must be available directly from the WhatsApp notification.
  • 5 Monthly challan summary report must be auto-generated and emailed to Ops Head on the 1st of each month.

Non-Functional Requirements

Out of Scope

  • ✕ Court challan disputes — handled by legal team separately, outside this system.
  • ✕ Challans for non-fleet vehicles (personal vehicles, vendor vehicles).
  • ✕ Integration with state-specific portals other than VAHAN (MoRTH).

Dependencies

GPS Module
Source of bus number plate data — required for VAHAN plate lookup queries
VAHAN API (MoRTH)
Government API for challan lookup by number plate — external dependency

Timeline

  • 15 April 2025

  • 30 April 2025
    Milestone 3 — Full rollout across all hubs

Open Questions

Open Question 2
