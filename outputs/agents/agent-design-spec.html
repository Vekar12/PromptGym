Agent Design Specification
Loan File Pre-checker Agent
Technical specification for automated IGF-stage loan file validation before Legal review
Version: 1.2 Status: Approved for Dev Owner: Legal Ops Team Last Updated: 12 March 2025 System: iLens / CRM Integration

Agent Overview

Agent Name
Loan File Pre-checker Agent
Purpose
Auto-validate loan application files at IGF stage before Legal review
Trigger Condition
New file uploaded to iLens with status "Ready for Legal"
Output
Validation report + Slack notification + CRM status update

Tools Available (4)

1 document_reader
Input Schema
{ file_id: string,
  pages: number[] }
Output Schema
{ text: string,
  confidence: float,
  missing_pages: number[] }
Use Case
Reads and OCR-processes uploaded documents from iLens storage.

Input Schema
{ documents: object[],
  checklist_version: string }
Output Schema
{ passed: boolean,
  score: float,
  failed_items: string[],
  warnings: string[] }
Use Case
Validates documents against RBI / internal checklist requirements.
3 slack_notifier
Input Schema
{ channel: string,
  message: string,
  priority: "high" | "normal" }
Output Schema
{ sent: boolean,
  timestamp: string }
Use Case
Sends internal Slack notifications to Legal or Sales channels.
4 crm_updater
Input Schema
{ file_id: string,
  status: string,
  notes: string }
Output Schema
{ updated: boolean,
  updated_at: string }

Updates iLens CRM with validation outcome and notes for audit trail.

Decision Flow

⏱ Trigger: New File in iLens
Status change detected: "Ready for Legal"
↓
📡 Fetch Documents
Tool: document_reader — reads all pages, computes confidence scores
↓
✓ Run Checklist Validation
Tool: checklist_validator — checks against current RBI/IGF checklist
↓
▶ Decision Gate: Score Threshold
Evaluated: score > 0.85 AND no critical failures
↓
✓ PASS — score > 0.85
Update CRM → "Legal Ready"
Notify Legal Team via Slack
Tool: crm_updater + slack_notifier
✕ FAIL — score ≤ 0.85
Update CRM → "Returned to Sales"
Notify Sales Manager with failed items
Tool: crm_updater + slack_notifier
↓
▮ END — Audit log written to iLens

Guardrails

⚠ Rule 1 — Document Integrity
Must NOT modify or delete any uploaded document. Agent is read-only with respect to iLens document storage.
⚠ Rule 2 — Confidence Threshold
Must NOT approve a file with confidence score < 0.80 on any critical document (KYC, income proof, property docs).
⚠ Rule 3 — Notification Scope
Must NOT send notifications to external parties. All Slack messages must target internal channels only (verified by channel whitelist).
⚠ Rule 4 — Checklist Currency
Must NOT proceed if checklist_version is outdated (>30 days old). Agent must halt and alert compliance team for checklist refresh.

Escalation Condition

🚨 Human-in-the-Loop Trigger

If document_reader confidence < 0.60 on any single page, the agent pauses immediately and routes the file to a human reviewer (Legal QA Team) via Slack. The notification includes: file ID, affected page numbers, and confidence scores for each flagged page. The agent does not proceed to checklist validation until human review is complete and a manual override is issued.
