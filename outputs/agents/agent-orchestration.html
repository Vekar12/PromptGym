Multi-Agent Orchestration Blueprint

Pattern: Hierarchical · Task: Automated Competitive Research Report

Execution Flow
O
Orchestrator Agent
Receives user query, decomposes into subtasks, assigns agents, collects results, synthesises final output. Never executes tools directly.
P
Planner Agent
Builds execution DAG from subtask list. Identifies parallelisable steps. Returns structured task graph with dependencies.
E
Executor Agents (×3 parallel)
Web search, competitor pricing scraper, LinkedIn signal tracker. Each runs independently, returns structured JSON to orchestrator.
C
Critic Agent
Validates executor outputs: checks for recency (≥2026), source credibility score ≥0.7, contradictions flagged for human review.
M
Memory Agent
Stores validated findings to vector DB. Retrieves prior reports for delta comparison. Prevents redundant searches within 24h window.

Guardrails
🚦 Executor agents cannot write to memory directly — all writes route through Critic validation first.
🚦 Max orchestration depth: 2 levels. Sub-agents cannot spawn further sub-agents.
🚦 Human-in-the-loop trigger: any finding with confidence <0.6 pauses the pipeline and surfaces for review.
