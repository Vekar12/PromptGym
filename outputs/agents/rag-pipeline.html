RAG Pipeline Architecture

Production-grade retrieval-augmented generation for internal knowledge base Q&A

Phase 1Ingestion
Document Loader
PDF, DOCX, Confluence, Notion via connectors. Incremental sync every 6h.
→
Chunker
Recursive character split. Chunk: 512 tokens, overlap: 64. Preserve section headers.
→
Embedder
text-embedding-3-large. 3072 dims. Batch 100 docs/call.
→
Vector Store
Pinecone (prod) / Chroma (dev). HNSW index. Metadata: source, dept, date.
Phase 2Retrieval
Query Rewriter
GPT-4o rewrites ambiguous queries. HyDE for sparse queries. Returns 3 variants.
→
Hybrid Search
Dense (cosine) + BM25 sparse. RRF fusion. Top-k: 20.
→
Reranker
Cohere rerank-v3. Cross-encoder scoring. Returns top 5 chunks.
→
Context Builder
Assemble context window. Inject source metadata. Max 4096 tokens.
Phase 3Generation
Prompt Assembly
System + context + user query. Role: internal assistant. Cite sources inline.
→
LLM Call
claude-3-5-sonnet. Temp: 0.2. Max tokens: 1024. Stream to UI.
→
Citation Validator
Verify each [Source X] exists in retrieved docs. Strip hallucinated cites.
